## Your first milestone: Foundations

Foundations is where every learner should begin. It gives you the vocabulary and habits needed for more advanced TypeScript—not just syntax to memorize.

| # | Lesson | Goal |
| ---: | --- | --- |
| 1 | [What is TypeScript?](./phase-1-foundations/lesson-01-what-is-typescript/) | Understand what TypeScript adds to JavaScript and where it fits in a real workflow. |
| 2 | [Installation, Compiler, and `tsconfig`](./phase-1-foundations/lesson-02-installation-compiler-tsconfig/) | Set up TypeScript and understand the compiler configuration you will use throughout the course. |
| 3 | [Primitive Types](./phase-1-foundations/lesson-03-primitive-types/) | Model simple values with clear, safe types. |
| 4 | [Arrays](./phase-1-foundations/lesson-04-arrays/) | Work confidently with typed collections. |
| 5 | [Objects and Type Aliases](./phase-1-foundations/lesson-05-objects-and-type-aliases-in-typescript/) | Describe data shapes that resemble real application data. |
| 6 | [Functions](./phase-1-foundations/lesson-06-functions-in-typescript/) | Type parameters, return values, and reusable behavior. |
| 7 | [Interfaces](./phase-1-foundations/lesson-07-interfaces-in-typescript/) | Design clear contracts for objects and application boundaries. |

After completing these lessons, you should be able to configure a TypeScript project, read type errors with more confidence, and model common application data without reaching for `any`.

## How to use each lesson

Use this sequence to get the most value from the course:

1. Read the lesson overview and identify the learning outcome.
2. Work through the notes and code examples slowly.
3. Attempt the exercises without looking at solutions.
4. Compare your approach with the worked solution.
5. Write down one mistake or insight in your own learning journal.
6. Move forward only when you can explain the concept in plain language.

> **Tip:** TypeScript is learned by writing and checking code. Reading a solution is useful only after you have made a genuine attempt.

## What comes next

Once you complete Foundations, continue into the [Intermediate phase](./phase-2-intermediate/). This phase introduces more expressive ways to model data and reusable behavior, including unions, intersections, generics, utility types, narrowing, and modules.

The complete learning sequence, planned outcomes, assessments, and milestone projects are documented in the [roadmap](./ROADMAP.md).
