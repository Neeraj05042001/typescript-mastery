# Milestone Projects

The milestone projects are where TypeScript Mastery moves from isolated concepts to production-style decisions. Each project is cumulative: it asks you to apply the types, boundaries, validation, and error-handling patterns introduced in earlier lessons.

> **Release status:** Projects will be published alongside the production and enterprise phases. Their folders will appear here only when they include runnable instructions and meaningful learning material.

| Project | Focus | Status |
| --- | --- | --- |
| **01 — Typed Todo API** | Express, CRUD, request and response contracts | Planned |
| **02 — Typed Express Application** | Middleware, validation, authentication, error handling | Planned |
| **03 — Full-Stack Typed Application** | React, Next.js, Node.js, shared types, production architecture | Planned |

## What every project will include

- A concise product brief and acceptance criteria
- Architecture and folder-structure explanation
- Type design decisions and API contracts
- Setup and run instructions
- Starter code or guided checkpoints where appropriate
- Validation, error handling, and testing considerations
- Trade-offs: what was deliberately chosen, deferred, or left out

The goal is not to copy an app. The goal is to learn how TypeScript improves the decisions that make an app reliable.
